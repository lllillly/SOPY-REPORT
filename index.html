SOPY PROJECT REPORT

STRUCTURE >>

  • DB: MongoDB -> graphql(ORM, ODM)로 BackEnd와 연결

  • BackEnd : NodeJS -> Graphql로 DB와 연결

  • FrontEnd : React -> Apollo(의 기능 중 하나)로 Graphql과 연결

    • ※ FrontEnd - req -> BackEnd // BackEnd - res -> FrontEnd


EXPLANATION OF TERMS >>

  • ORM : 객체(Class) - ORM - DB 데이터

    객체 지향 프로그래밍 : 클래스 사용 / 관계형 데이터 베이스 : 테이블을 사용

    위의 구조를 이유로 객체 모델과 관계형 모델간에 불일치가 존재하여, 이 객체간의 관계를 바탕으로 SQL을 자동 생성하여 불일치를 해결한다.

  • ODM : NoSQL에서 Document Database를 지원하기 위해 데이터를 변환하는 프로그래밍 기법을 일컫는다.

    ⌜NOSQL : 관계형 데이터베이스 뿐만 아니라 다양한 데이터베이스를 명칭 , 빅데이터를 다룰 때 RDBMS로만 트래픽을 감당하기 어려워져 이를 해결하기 위해 탄생하였다.⌟

  • APOLLO CLIENT : GraphQL로 로컬 및 원격 데이터를 모두 관리 할 수있는 JavaScript 용 포괄적 인 상태 관리 라이브러리, UI를 자동으로 업데이트하고, 애플리케이션 데이터를 가져오고, 캐시할 수 있도록 한다.
